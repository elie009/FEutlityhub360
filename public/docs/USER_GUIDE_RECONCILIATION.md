Learn how to reconcile your bank accounts with bank statements to ensure your records match your bank's records.

## üìö What is Bank Reconciliation?

Bank reconciliation is the process of matching transactions in your UtilityHub360 account with transactions on your bank statement. This helps you:
- **Verify Accuracy**: Ensure all transactions are recorded correctly
- **Find Discrepancies**: Identify missing or duplicate transactions
- **Maintain Accuracy**: Keep your account balances accurate
- **Detect Errors**: Catch bank errors or unauthorized transactions

## üöÄ Accessing Reconciliation

1. Go to **Finance ‚Üí Bank Accounts** in the sidebar
2. Select the bank account you want to reconcile
3. Click **"Reconcile"** or navigate to the **Reconciliation** page
4. You'll see your reconciliation dashboard

## üìä Understanding the Reconciliation Dashboard

### Summary Cards
- **Book Balance**: Your account balance in UtilityHub360
- **Statement Balance**: Balance from your bank statement
- **Difference**: The difference between book and statement balance
- **Matched Transactions**: Number of transactions successfully matched
- **Unmatched Transactions**: Transactions that need review

### Reconciliation Status
- **PENDING**: Reconciliation in progress
- **COMPLETED**: All transactions matched and reconciled
- **DISCREPANCY**: Differences found that need attention

## üì• Importing Bank Statements

### Step 1: Import Statement
1. Click **"Import Statement"** button
2. Choose your import method:
   - **Upload File**: Upload CSV, OFX, or QIF file
   - **Manual Entry**: Enter transactions manually
   - **PDF Upload**: Extract data from PDF statement

### Step 2: Configure Statement Details
Fill in the required information:
- **Statement Name**: e.g., "January 2024 Statement"
- **Statement Period**: Start and end dates
- **Opening Balance**: Balance at start of period
- **Closing Balance**: Balance at end of period
- **Account**: Select the bank account

### Step 3: Review Imported Transactions
- Review all imported transactions
- Verify dates, amounts, and descriptions
- Edit or delete incorrect entries
- Add missing transactions manually

### Step 4: Save Statement
Click **"Save Statement"** to store the imported statement

## üîó Matching Transactions

### Automatic Matching
The system automatically matches transactions based on:
- **Amount**: Exact or near-exact matches
- **Date**: Transactions within a date range
- **Description**: Similar transaction descriptions
- **Reference Number**: Matching reference numbers

### Manual Matching
1. Click on a reconciliation to open the matching view
2. Review **Unmatched Transactions** section
3. For each unmatched transaction:
   - Review suggested matches (if any)
   - Select the correct match from suggestions
   - Or manually link to a statement item
4. Click **"Match"** to confirm

### Match Types
- **AUTO**: Automatically matched by the system
- **MANUAL**: Manually matched by you
- **UNMATCHED**: Not yet matched

## ‚úÖ Completing Reconciliation

### Step 1: Review All Matches
- Check that all transactions are matched correctly
- Verify amounts and dates are accurate
- Review any discrepancies

### Step 2: Resolve Discrepancies
If there are differences:
- **Missing Transactions**: Add transactions that appear on statement but not in system
- **Extra Transactions**: Review transactions in system but not on statement
- **Amount Differences**: Verify and correct amount discrepancies

### Step 3: Finalize Reconciliation
1. Review the reconciliation summary
2. Ensure difference is zero (or explain any remaining difference)
3. Add notes if needed
4. Click **"Complete Reconciliation"**

## üõ†Ô∏è Troubleshooting

### Common Issues

#### Difference Not Zero
- **Check for Missing Transactions**: Look for transactions on statement not in system
- **Verify Amounts**: Double-check transaction amounts
- **Review Dates**: Ensure transaction dates are correct
- **Check for Duplicates**: Look for duplicate transactions

#### Transactions Won't Match
- **Verify Amounts**: Ensure amounts match exactly (or within tolerance)
- **Check Dates**: Transaction dates should be close
- **Review Descriptions**: Descriptions should be similar
- **Manual Match**: Use manual matching if automatic matching fails

#### Statement Import Errors
- **Check File Format**: Ensure file is CSV, OFX, or QIF format
- **Verify Data**: Check that all required fields are present
- **Review Encoding**: Ensure file uses correct character encoding
- **Try Manual Entry**: Use manual entry if import fails

## ‚ùì Frequently Asked Questions

### How often should I reconcile?
We recommend reconciling monthly, or more frequently if you have many transactions.

### What if I can't find a match?
You can manually add the transaction or mark it as unmatched for later review.

### Can I reconcile multiple accounts?
Yes, you can reconcile each bank account separately.

### What happens to unmatched transactions?
Unmatched transactions remain in the system for future reconciliation or manual review.

### Can I undo a reconciliation?
Yes, you can reopen a completed reconciliation if needed.
